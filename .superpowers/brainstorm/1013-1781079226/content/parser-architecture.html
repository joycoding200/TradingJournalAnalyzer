解析器插件架构

插件式，新增券商只需一个文件

统一接口

BaseParser — 所有解析器的抽象基类
class BaseParser(ABC):
    """所有券商解析器必须实现的接口"""

    @classmethod
    @abstractmethod
    def source_type(cls) -> str:
        """标识符，如 'qmt', 'vnpy', 'dfcf', 'ths', 'wenhua', 'boyi', 'ctp'"""
        ...

    @classmethod
    @abstractmethod
    def asset_type(cls) -> str:
        """资产类别 'stock' | 'future'"""
        ...

    @classmethod
    @abstractmethod
    def detect(cls, file: UploadedFile) -> float:
        """检测此文件是否为该格式，返回置信度 0.0~1.0"""
        ...

    @classmethod
    @abstractmethod
    def parse(cls, file: UploadedFile, raw_file_id: UUID, user_id: UUID) -> list[Trade]:
        """解析文件，返回 Trade 列表"""
        ...

ParserRegistry 自动发现

注册表 — 自动扫描所有 Parser 子类
class ParserRegistry:
    """自动发现并注册所有 BaseParser 子类"""

    _parsers: list[type[BaseParser]] = []

    @classmethod
    def auto_discover(cls):
        """扫描 parsers/ 目录，注册所有 BaseParser 子类"""

    @classmethod
    def detect_format(cls, file) -> list[tuple[str, float]]:
        """返回所有匹配的 (source_type, confidence)，按置信度降序"""

    @classmethod
    def get_parser(cls, source_type: str) -> type[BaseParser]:
        """按标识符获取对应解析器"""

    @classmethod
    def parse(cls, source_type: str, file, raw_file_id, user_id) -> list[Trade]:
        """委托给对应解析器"""

解析流程

上传文件 → Trade 列表
① 用户上传文件 → 保存 RawFile，拿到 raw_file_id
② ParserRegistry.detect_format() → 所有解析器各返回一个置信度
例如：QMT: 0.95, 东方财富: 0.12, VN.PY: 0.08 → 选 QMT
③ 如果最高置信度 < 阈值（如 0.7）→ 返回候选列表让用户手动选择
④ ParserRegistry.parse(source_type, ...) → 得到 Trade[] 列表
⑤ Trade[] 批量写入 PostgreSQL → 返回给前端预览

detect() 策略

解析器 检测方法
QMT列名匹配 "委托时间/成交时间/证券代码/买卖方向"
VN.PY列名匹配 "datetime/symbol/direction/price/volume"
东方财富列名匹配 "成交日期/证券代码/操作/成交均价"
同花顺列名匹配 "发生日期/证券代码/买卖标志/成交价格"
文华财经列名匹配 "开平/合约/手数/成交价" + asset_type=future
博易大师列名匹配 "成交日期/合约/买卖/成交价/手数"
CTP/快期/易盛列名匹配 "交易日/合约代码/买卖/成交量/成交价"
华泰/中信/国君...列名匹配中文表头 + 券商特有字段模式

💡 detect() 的核心思路：打开文件 → 读表头 → 与各券商模板比对列名 → 打分 → 返回最高者。 不靠文件名、不靠魔数，纯表头匹配。
