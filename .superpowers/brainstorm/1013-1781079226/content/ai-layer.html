Prompt 模板结构

三段式结构化 Prompt
SYSTEM (角色约束)
你是一名 A 股交易教练。你的职责是分析用户的交易数据，帮助他识别亏损原因并给出改善建议。
你必须严格遵守以下规则：
1. 只使用我提供的数据，不要编造任何数字
2. 不要预测市场走势
3. 不要推荐具体股票
4. 分析要基于行为，不要归因于市场或运气
5. 语气：专业、直接、不说废话
USER (数据 + 任务)
以下是一位交易者的账户分析数据：

{{ structured_json_here }}

请从以下四个维度生成诊断报告：
1. 你的优势（最赚钱的行为模式是什么？）
2. 你的劣势（最亏钱的行为模式是什么？）
3. 最危险行为（What If 回测显示，删除哪种行为收益提升最大？）
4. 改善建议（具体的、可执行的下一步行动）

验证层

AI 输出 → 正则提取数字 → 与输入 JSON 对比 → 偏差超阈值则重试
class ReportValidator:
    """验证 AI 生成的报告中的数字是否准确"""

    def validate(self, report: str, input_data: dict) -> ValidationResult:
        # 从报告中提取所有数字（百分比、金额、次数）
        report_numbers = extract_numbers(report)

        # 从输入 JSON 中提取对应的参考值
        reference_numbers = extract_reference_values(input_data)

        # 逐一比对
        errors = []
        for num in report_numbers:
            if not matches_reference(num, reference_numbers, tolerance=0.01):
                errors.append(f"报告说 {num}，但实际数据是 {reference_numbers[...]}")

        return ValidationResult(
            passed=len(errors) == 0,
            errors=errors
        )

    # 如果未通过验证 → 重试，最多 3 次
    # 3 次仍失败 → 返回不带数字的通用报告 + 标记 validation_failed

Provider 抽象

class LLMProvider(ABC):
    """环境变量 AI_PROVIDER 决定用哪个实现"""
    @abstractmethod
    async def generate(self, system_prompt: str, user_prompt: str) -> str: ...

class OpenAIProvider(LLMProvider): ...
class ClaudeProvider(LLMProvider): ...
class DeepSeekProvider(LLMProvider): ...

# 工厂函数
def get_llm() -> LLMProvider:
    provider = os.getenv("AI_PROVIDER", "openai")  # openai | claude | deepseek
    return {"openai": OpenAIProvider, "claude": ClaudeProvider, "deepseek": DeepSeekProvider}[provider]()
