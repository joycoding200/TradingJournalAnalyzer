 TradingJournalAnalyzer 环境配置总结

  一、开发环境

1. 基础设施依赖
   ┌───────────────┬────────┬───────────────────────────────────────────────────────────────────┐
   │     组件      │  要求  │                               说明                                │
   ├───────────────┼────────┼───────────────────────────────────────────────────────────────────┤
   │ PostgreSQL 17 │ 必须   │ 数据库，连接串在 .env 中配置                                      │
   ├───────────────┼────────┼───────────────────────────────────────────────────────────────────┤
   │ Python 3.x    │ 必须   │ 后端运行环境                                                      │
   ├───────────────┼────────┼───────────────────────────────────────────────────────────────────┤
   │ Node.js + npm │ 必须   │ 前端构建工具链                                                    │
   ├───────────────┼────────┼───────────────────────────────────────────────────────────────────┤
   │ 虚拟环境      │ 必须   │ backend/.venv/，用 source .venv/bin/activate 激活                 │
   ├───────────────┼────────┼───────────────────────────────────────────────────────────────────┤
   │ mootdx        │ 运行时 │ 通达信行情数据（TCP 7709），首次需 python -m mootdx bestip 初始化 │
   └───────────────┴────────┴───────────────────────────────────────────────────────────────────┘

2. 后端环境变量 (backend/.env)
   从 backend/.env.example 复制，核心配置：
   
   # 数据库 — 开发用本地 PostgreSQL
   
   DATABASE_URL=postgresql://postgres:postgres@localhost:5432/tradelens
   
   # JWT — 开发环境随意填
   
   SECRET_KEY=dev-only-change-this-to-a-random-32-char-string
   JWT_ALGORITHM=HS256
   JWT_EXPIRE_MINUTES=480
   
   # AI — 选一个提供商填 key
   
   AI_PROVIDER=openai              # openai | claude | deepseek | openrouter
   OPENAI_API_KEY=sk-xxx
   OPENAI_MODEL=gpt-4o-mini
   
   # CORS — 前端 dev server 地址
   
   CORS_ORIGINS=http://localhost:5173
   
   # 环境标识
   
   ENV=development

3. 前端环境变量 (frontend/.env)
   VITE_API_BASE=http://localhost:8000   # 后端 API 地址

4. 启动命令
   
   # 后端（端口 8000）
   
   cd backend
   source .venv/bin/activate
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   
   # 前端（端口 5173，通过 vite proxy 转发 /api 到 8000）
   
   cd frontend
   npm install
   npm run dev
   
   # 测试
   
   cd backend && pytest tests/ -q

  二、生产环境

  项目目前的目标部署架构是 阿里云 ECS + Nginx + systemd（见 deploy/README.md）。

1. 基础设施
   ┌─────────────────┬─────────────────────────────────────────────┐
   │      组件       │                    说明                     │
   ├─────────────────┼─────────────────────────────────────────────┤
   │ 阿里云 ECS      │ 运行服务器                                  │
   ├─────────────────┼─────────────────────────────────────────────┤
   │ PostgreSQL 17   │ 裸机安装                                    │
   ├─────────────────┼─────────────────────────────────────────────┤
   │ Nginx           │ 80 端口统一入口：静态文件 + /api 反代到后端 │
   ├─────────────────┼─────────────────────────────────────────────┤
   │ systemd         │ 管理 FastAPI 服务（服务名 tja-backend）     │
   ├─────────────────┼─────────────────────────────────────────────┤
   │ Python 虚拟环境 │ backend/.venv/                              │
   └─────────────────┴─────────────────────────────────────────────┘

2. 后端环境变量 — 生产强制要求
   与开发环境的 关键差异（deploy/README.md 中有对照表）：
   ┌──────────────────┬─────────────────────────────────────────────────────────┬─────────────────────────────────┐
   │      配置项      │                        开发环境                         │            生产环境             │
   ├──────────────────┼─────────────────────────────────────────────────────────┼─────────────────────────────────┤
   │ ENV              │ development                                             │ production（触发严格校验）      │
   ├──────────────────┼─────────────────────────────────────────────────────────┼─────────────────────────────────┤
   │ DATABASE_URL     │ postgresql://postgres:postgres@localhost:5432/tradelens │ 生产密码，正式连接串            │
   ├──────────────────┼─────────────────────────────────────────────────────────┼─────────────────────────────────┤
   │ SECRET_KEY       │ 随意                                                    │ ≥32 字符随机串（openssl rand    │
   │                  │                                                         │ -hex 32），否则启动报错         │
   ├──────────────────┼─────────────────────────────────────────────────────────┼─────────────────────────────────┤
   │ CORS_ORIGINS     │ http://localhost:5173                                   │ 实际域名或 IP，如               │
   │                  │                                                         │ http://47.xxx.xxx.xxx           │
   ├──────────────────┼─────────────────────────────────────────────────────────┼─────────────────────────────────┤
   │ AI_PROVIDER      │ openai                                                  │ deepseek（推荐，国内速度快）    │
   ├──────────────────┼─────────────────────────────────────────────────────────┼─────────────────────────────────┤
   │ DEEPSEEK_API_KEY │ （空）                                                  │ 实际 API Key                    │
   └──────────────────┴─────────────────────────────────────────────────────────┴─────────────────────────────────┘
   ▎ SECRET_KEY 生产强制：config.py 中有 validator，ENV=production 时不允许使用默认值、且长度必须 ≥32 字符，否则
   ▎ RuntimeError。

3. 前端环境变量
   VITE_API_BASE=   # 空字符串或同域（Nginx 反代，同域部署时留空即可）
   构建：npm run build → dist/ 由 Nginx 托管。

4. 部署流程
   三种方式（deploy/ 目录）：
   ┌────────────────┬──────────────────────────────────────┬─────────────────────────────────────┐
   │      方式      │                 命令                 │              适用场景               │
   ├────────────────┼──────────────────────────────────────┼─────────────────────────────────────┤
   │ 本地一键       │ bash deploy/local-push.sh            │ 日常推荐，push + SSH 触发服务器更新 │
   ├────────────────┼──────────────────────────────────────┼─────────────────────────────────────┤
   │ 手动 SSH       │ ssh 到服务器 → bash deploy/update.sh │ 需要手动控制                        │
   ├────────────────┼──────────────────────────────────────┼─────────────────────────────────────┤
   │ GitHub Actions │ push main 自动触发                   │ 全自动 CI/CD                        │
   └────────────────┴──────────────────────────────────────┴─────────────────────────────────────┘
   update.sh 执行流程：git pull → npm build → pip install → alembic upgrade head → systemctl restart → 健康检查

5. 生产配置校验
   部署后在服务器运行：
   bash deploy/config-check.sh
   自动检查 6 个维度：.env 存在性、关键配置项、AI 提供商、数据库连接、管理员账户、systemd 服务状态。

6. 其他生产注意事项
- 数据库迁移：生产环境必须用 alembic upgrade head，不能用 create_all（main.py 注释中有说明）
- HTTPS：有域名时配 Let's Encrypt + certbot，同时改 jwt.py 中 secure=True
- uploads/ 目录：用户上传的交割单文件存在 backend/uploads/，.gitignore 已排除，不会被部署覆盖
- 管理员账户：通过 deploy/init-admin.sh 创建
- 数据库备份：建议 crontab 定时 pg_dump
